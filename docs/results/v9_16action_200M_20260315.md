# V9 16-Action Grid Experiment — 200M Iterations (2026-03-15)

## Hypothesis

Adding 3 new postflop bet sizes (BET_QUARTER_POT 0.25x, BET_THREE_QUARTER_POT 0.75x, BET_DOUBLE_POT 2.0x) would improve performance against WeirdSizingBot and reduce off-tree translation errors, while maintaining or improving overall gauntlet performance.

## Configuration

- **Action grid:** 16 actions (v9 expanded — 3 new postflop sizes)
- **Phase schedule:** 2x (same as B0)
- **All-in dampening:** old (rc<2, 0.7x — same as B0)
- **Dynamics:** Standard CFR+ (no DCFR)
- **Workers:** 6 parallel
- **Training:** 100M fresh + 100M continued = 200M total

```bash
# Phase 1: Fresh 100M
venv/bin/python train_gto.py 100000000 --workers 6 --fresh \
  --phase-schedule 2x --allin-dampen old \
  --checkpoint-interval 10000000 --checkpoint-eval --checkpoint-gauntlet

# Phase 2: Continue to 200M
venv/bin/python train_gto.py 100000000 --workers 6 \
  --phase-schedule 2x --allin-dampen old \
  --checkpoint-interval 10000000 --checkpoint-eval \
  --checkpoint-gauntlet --gauntlet-interval 50000000
```

## Analysis

### Result: NEGATIVE — 16-action grid underperforms B0

The expanded action grid did NOT converge at 200M iterations despite having per-node parity with B0 (100 iters/node vs 95). Exploitability plateaued at ~40 bb/100, roughly 33x worse than B0's 1.22.

### Why exploitability plateaued instead of converging

The per-node iteration budget assumes linear scaling, but the actual relationship is worse than linear:

1. **Combinatorial explosion in action sequences.** Each postflop node has 10-11 actions instead of 7-8. A 3-action betting sequence has 10^3 = 1000 paths vs 7^3 = 343 — nearly 3x more paths to explore per street.

2. **Adjacent size confusion.** The solver struggles to differentiate between adjacent sizes: quarter (0.25) vs third (0.33), two-thirds (0.67) vs three-quarter (0.75), overbet (1.25) vs double-pot (2.0). These pairs have similar EV, so regrets stay near-zero and strategies remain flat/uniform.

3. **Deeper game trees.** More actions per node means deeper re-raise sequences before the depth limit is hit, creating more nodes at depth that are rarely visited.

### Per-bot diagnosis

**Persistently negative (structural problem):**
- **NitBot (-1289):** The solver has too many bet sizes to choose from and picks poorly against a tight player. When NitBot calls, it has a strong hand, and our random sizing over-commits.
- **PerturbBot (-13):** Near-random strategy amplifies PerturbBot's noise instead of absorbing it.

**Positive but weaker than B0:**
- **CallStationBot (+705 vs +1071):** Value betting works but is diluted across too many sizes — the solver hasn't learned to concentrate value bets on optimal sizes.
- **OverfoldBot (+32 vs +116):** Can't efficiently exploit folds because bet frequency is spread across 9 sizes instead of 6.

**Improved over B0:**
- **AggroBot (+1527 vs +893):** More sizing options help counter hyper-aggression — this is the one matchup where the wider grid adds genuine strategic value.
- **DonkBot (+1047 vs +1024):** Marginal improvement.

**Unchanged:**
- **WeirdSizingBot (+218 vs +220):** The primary motivation for adding sizes. Zero improvement. The new sizes don't help because the *solver's* sizing is unconverged, not just the translation.

### DCFR sweep results (also negative)

Tested gamma = {0.999, 0.995, 0.99} at 20M iterations on the 16-action grid. All variants showed exploitability *increasing* from 10M to 20M — DCFR's regret discounting destabilizes convergence on wider action trees.

| Gamma | 10M expl | 20M expl | 20M gauntlet |
|-------|---------|---------|-------------|
| 0.999 | 4.37 | 8.60 | -62.9 |
| 0.995 | 6.90 | 12.99 | +411.5 |
| 0.99 | — | — | (not run) |

DCFR is counterproductive for this grid size.

## Conclusion

The 16-action grid expansion (Phase 3A) is **not viable at current iteration budgets**. The wider action tree creates a convergence problem that can't be solved by simply training longer — exploitability plateaued at ~40 bb/100 rather than continuing to decrease.

### Recommendation

1. **Revert to 13-action grid for production.** B0 (expl 1.22, gauntlet +677) remains the best strategy.
2. **If pursuing wider grids in the future**, consider:
   - Adding only 1 size at a time (e.g. just BET_THREE_QUARTER_POT which fills the biggest gap)
   - Phase-gated expansion: add sizes only on river where translation gaps matter most
   - Much longer training budgets (500M+) or warm-starting from the converged 13-action strategy
